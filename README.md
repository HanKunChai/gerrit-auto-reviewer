# GERA — Gerrit Auto-Review

> 基于 Claude Code CLI 的 Gerrit 自动代码评审系统，专为内网嵌入式 C 项目设计。

---

## 项目评估

**GERA 有实际意义**，其核心价值体现在以下几点：

| 价值点 | 说明 |
|--------|------|
| 解决真实痛点 | 代码评审是团队开发的瓶颈之一，自动化初审可以将人工 reviewer 从重复性检查中解放出来 |
| 内网可用 | 仅依赖 `flask`、`pyyaml`、`requests` 三个包，Claude Code CLI 本地运行，**零外部 API 调用**，满足企业内网隔离要求 |
| 双引擎互补 | 正则规则引擎负责确定性检查（空指针、strcpy、缓冲区溢出等），Claude Code 负责语义理解，两者结果合并上报 |
| 工程质量可靠 | 幂等设计（同一变更同一版本不重复评审）、并发安全（独立 `refs/review/*` 命名空间）、失败静默（Claude 出错不上报 Gerrit） |
| 可扩展性 | 提示词和规则均为外部文件，新增语言只需增加 prompt 模板和 YAML 规则 |

**主要局限**：依赖 Claude Code CLI 已安装并完成认证；当前规则和提示词仅覆盖嵌入式 C 语言；评审分数固定为 0（只发表评论，不自动通过/拒绝）。

## 系统架构

```
Gerrit REST API
      │
      ├─── auto_review_poller.py   ← 主入口，持续轮询
      │         │
      │    ThreadPoolExecutor (4 workers)
      │         │
      │    _review_change()
      │         ├─ GerritClient        → 获取变更信息、提交评审
      │         ├─ LocalRepo           → git shallow clone + fetch change diff
      │         ├─ ReviewEngine        → 正则规则匹配 (可选)
      │         └─ Claude Code CLI     → AI 语义评审 (subprocess)
      │
      └─── mcp_gerrit_server/      ← MCP 服务模式（供 Claude Code MCP 客户端使用）
```

**两种运行模式**（互斥）：

- **轮询模式**：`auto_review_poller.py` 自主运行，定时查询 Gerrit → 评审 → 提交，无需外部触发
- **MCP 服务模式**：`server.py` 作为 MCP Server，由 Claude Code 等客户端主动调用工具执行评审

## 快速开始

### 前置条件

- Python 3.8+
- Claude Code CLI（`claude` 命令可用，已登录）
- 可访问的 Gerrit 实例，拥有 `code-reviewer` 服务账号

### 安装

```bash
git clone <this-repo>
cd gerrit-auto-reviewer

# 安装依赖
# 方式 1：有外网或内网 PyPI 镜像
# 直接把项目目录拷到目标 Linux 机器，然后
bash install.sh

# 方式 2：纯内网无镜像（离线部署）


# 在外网 Linux 机器上先下载依赖
bash scripts/download_deps.sh    # 生成 offline-packages/

# 把整个项目目录（含 offline-packages/）拷到内网机器
# 然后在内网运行
bash install.sh
# install.sh 会自动检测 offline-packages/ 目录进行离线安装。
```

### 配置

编辑 `config.yaml`，最少需要修改以下字段：

```yaml
gerrit:
  base_url: "http://gerrit.your.company.com:8080"
  auth:
    username: "code-reviewer"    # Gerrit 服务账号用户名
# 密码在 .env 文件中设置: GERRIT_PASSWORD=xxx

auto_review:
  reviewer: "code-reviewer"     # 与 username 保持一致
  query: "reviewer:code-reviewer+status:open"
```

编辑 `.env`

```env
GERRIT_PASSWORD=your_http_password_here
```

### 运行

```bash
# 测试连通性（Mock 模式，不连真实 Gerrit）
python scripts/auto_review_poller.py --mock --once --verbose

# 单次执行（连接真实 Gerrit，跑完一轮即退出）
python scripts/auto_review_poller.py --once --verbose

# 持续后台运行（推荐生产使用）
bash run.sh #被动模式

# 启动轮询评审
bash run.sh --poller

# 或只跑一轮测试
bash run.sh --poller --once

# 或
python scripts/auto_review_poller.py
```

## 评审流程

```
1. 查询 Gerrit: GET /changes/?q=reviewer:code-reviewer+status:open
2. 过滤: 跳过自己的提交 / 已在评审中 / 已评审过当前版本
3. 并发: ThreadPoolExecutor 异步处理（默认 4 线程）
4. 获取 diff: git fetch refs/changes/{xx}/{id}/{rev} → 本地 refs/review/{id}/{rev}
5. 规则引擎: 正则扫描新增行（可配置开关）
6. Claude Code: subprocess 调用，在仓库目录执行，最多 10 轮，超时 30 分钟
7. 提交评审: POST /changes/{id}/revisions/{rev}/review（score=0，tag 标记版本）
8. 清理: 删除临时 refs/review/* ref
```

**幂等保证**：通过 `autogenerated:gerrit:auto-review:v4` tag + account_id + revision_number 三要素判断是否已评审，同一 patch set 不会重复提交。

## 规则引擎

内置 17 条 C 语言规则（默认关闭，在 `config.yaml` 中设置 `use_rules_engine: true` 启用）：

| 类别 | 规则数 | 示例 |
|------|--------|------|
| 安全 | 6 条 | `strcpy`/`strcat`/`sprintf`/`gets` 缓冲区溢出，`system()` 注入 |
| 编码规范 | 5 条 | Allman 大括号风格，行长超 200，单行多变量 |
| 常见缺陷 | 6 条 | 空指针解引用，use-after-free，内存泄漏，除零风险 |

**自定义规则**：在 `review_rules/custom/` 目录下添加 YAML 文件，格式参考 `review_rules/builtin/*.yaml`。

## MCP 服务模式

若使用 Claude Code MCP 客户端（而非轮询），启动 MCP Server：

```bash
python -m mcp_gerrit_server.server [--mock] [--verbose]
```

提供 7 个 MCP 工具：`list_changes`、`fetch_diff`、`get_file_context`、`post_review`、`repo_status`、`run_rules`、`c_review_prompt`。

## 配置参考

```yaml
auto_review:
  poll_interval: 600          # 轮询间隔（秒），默认 10 分钟
  max_changes_per_poll: 10    # 每轮最多处理变更数
  use_rules_engine: false     # 是否启用正则规则引擎
  max_concurrent_reviews: 4   # 并发评审线程数
  claude_timeout: 1800        # Claude CLI 超时（秒），默认 30 分钟

repo:
  initial_clone_depth: 10     # 首次 clone 深度
  auto_deepen: true           # 遇到 shallow 不足时自动加深

cache:
  ttl_hours: 24               # 评审缓存有效期
```

## 常见问题

**Q: Claude 评审失败了怎么办？**  
A: 失败时系统只写日志，不向 Gerrit 上报任何信息，该变更在下一轮轮询时会被重新处理。

**Q: 如何诊断 Gerrit 认证问题？**  
```bash
python scripts/diagnose_auth.py --verbose
```

**Q: 如何只评审某个特定变更？**  
```bash
python scripts/auto_review.py --change <change_id>
```

**Q: 如何更新评审版本（让已评审的变更重新被评审）？**  
修改 `scripts/auto_review_poller.py` 中的 `REVIEWER_VERSION`（如 `"v4"` → `"v5"`），重启服务即可。

## 已知限制

- 规则引擎仅支持 C 语言；如需其他语言，需添加对应规则文件和提示词模板
- 评审分数固定为 `0`（不自动给 +1/+2 或 -1/-2）
- 依赖 Claude Code CLI 本地安装及登录状态
- `--dangerously-skip-permissions` 标志适用于内网受控环境，公网部署需评估风险
